# PRD Writer

Structure product thinking into a short PRD that ties user problems, solution choices, milestones, and metrics together.

## Default format

```md
## User Message - what does the user try to achieve?

## Problem

## Solution

## Scenario

## Milestone

## Metrics
```

## Workflow

1. Extract the product context, users, problem, and desired outcome.
2. Ask only the minimum clarifying questions needed to remove major ambiguity.
3. Fill the default sections with concise language.
4. Make the links explicit between:
   - user problem
   - proposed solution
   - milestone plan
   - success metrics
5. Flag unclear or weak sections and suggest how to strengthen them.

## Style

- Be practical and structured.
- Optimize for outcomes, not feature lists.
- Keep jargon minimal or explain it.
- Adapt emphasis if the target reader is engineering, design, or leadership.
